# Terminology Governance

This document ensures that the same concept is not described differently across pages without explanation.

## Terminology Rules

- use the canonical service names: Frontend UI, Backend API, Scraper API, and Model API
- use one preferred term for a core concept unless a variant is explicitly documented
- update the glossary or reference pages when a new cross-service term becomes important

## Canonical Sources

| Term Type                     | Canonical Source                           |
| ----------------------------- | ------------------------------------------ |
| Product and user-facing terms | Overview pages and glossary                |
| Core domain entities          | Reference pages                            |
| Service names and boundaries  | System landscape and service landing pages |
| Governance terms              | Standards pages                            |

## Change Rule

If a term changes:

1. update the canonical source page
2. update affected linked pages
3. avoid leaving mixed terminology without an explicit note

## Examples Of Terms That Must Stay Stable

- fit score
- skill gap analysis
- application tracker
- synchronized docs
- service landing page
