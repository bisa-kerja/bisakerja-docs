# Visual Asset Policy

This document governs diagrams, screenshots, and other visual assets used in the documentation system.

## Policy Goals

- Avoid stale visuals that contradict current system behavior
- Keep asset ownership and naming clear
- Prevent collisions during synchronization and versioning

## Diagram Rules

- Diagrams should live near the page that owns them unless intentionally shared
- Diagrams must use descriptive names with service or domain prefixes
- Architecture diagrams should be updated whenever system boundaries materially change

## Screenshot Rules

- Screenshots should be used only when they add real explanatory value
- Screenshots that frequently go stale should be replaced by structured text or diagrams when possible
- Screenshots should be versioned or refreshed when the UI changes materially

## Generated Asset Rules

- Generated visuals such as exported diagrams or generated references should be identified clearly
- Pipelines should avoid publishing assets with ambiguous names

## Stale Asset Handling

If a visual asset no longer reflects the system:

1. Update it
2. Replace it with a clearer artifact
3. Remove it if the same information is already covered better in text
