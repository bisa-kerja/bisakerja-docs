# Documentation Standards

The standards section defines how this repository is structured, how documentation is owned, and how changes are reviewed.

These rules exist so the central docs platform can scale without turning into a disconnected collection of pages copied from multiple repositories.

- [Information Architecture](./information-architecture.mdx)

- [Metadata Standard](./metadata-standard.mdx)
- [Contribution Guide](./contribution-guide.mdx)
- [Service Documentation Integration](./service-documentation-integration.mdx)
- [Documentation Sync and Versioning](./documentation-sync-and-versioning.mdx)

## Working Rule

If a document changes the central structure, ownership model, or cross-service understanding of the platform, it should align with the standards in this section before it is merged.
