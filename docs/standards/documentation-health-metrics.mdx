# Documentation Health Metrics

This document defines the metrics used to understand whether the documentation system is staying trustworthy over time.

## Core Metrics

| Metric                 | What It Measures                                                   |
| ---------------------- | ------------------------------------------------------------------ |
| Coverage               | Whether required docs exist for key sections and services          |
| Freshness              | Whether pages are still within expected review cadence             |
| Ownership completeness | Whether pages have clear owners and reviewers                      |
| Broken links           | Whether readers can navigate successfully                          |
| Sync health            | Whether synchronized docs are publishing and reconciling correctly |
| Stale asset count      | Whether visuals are lagging behind system changes                  |

## Suggested Operational Thresholds

- No broken links in the published site
- No ownerless active pages
- No critical landing page older than the expected review cadence without an explicit exception
- No failing sync health for core services without open follow-up

## How To Use The Metrics

- Use them to spot drift, not only to produce dashboards
- Review them during governance or docs maintenance cycles
- Escalate when structural or ownership problems persist across review periods
