# Services

The services section is the landing area for service-owned documentation that will later be synchronized from the individual repositories.

Phase 1 establishes the structure and ownership boundaries. Later phases will populate these sections with architecture, API, and operational content.

## Service Sections

- [Frontend UI](./frontend-ui/index.mdx)
- [Backend API](./backend-api/index.mdx)
- [Scraper API](./scraper-api/index.mdx)
- [Model API](./model-api/index.mdx)

## Structure Rule

Each service directory contains:

- a locally maintained `index.mdx` page for service summary and navigation
- a `synced/` directory reserved for imported service documentation
