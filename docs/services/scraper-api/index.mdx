# Scraper API

The Scraper API is the ingestion service responsible for collecting job data from external platforms and transforming it into normalized records that the rest of the Bisakerja platform can use.

## Source Repository And Sync Target

- Assumed source repository: `scraper-api`
- Source docs root: `/docs/**`
- Central sync target: `docs/services/scraper-api/synced/**`

## Platform Responsibilities

- fetch job data from external job platforms
- normalize heterogeneous source data into a stable internal shape
- support recurring refresh of job inventory and related metadata
- provide the raw-to-normalized bridge that powers search and recommendations

## Key Dependencies And Consumers

| Type                  | System                 | Why It Matters                                             |
| --------------------- | ---------------------- | ---------------------------------------------------------- |
| Upstream source       | External job platforms | Provide raw listings and platform-specific formats         |
| Downstream dependency | PostgreSQL             | Stores normalized job records                              |
| Downstream consumer   | Backend API            | Reads normalized jobs for user-facing search and retrieval |
| Cross-service context | Data Flow docs         | Explains how ingestion becomes usable platform data        |
