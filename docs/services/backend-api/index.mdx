# Backend API

The Backend API is the primary application backend for Bisakerja. It owns authentication, business logic, persistence, and orchestration between the frontend, the data ingestion layer, and the model-serving layer.

## Source Repository And Sync Target

- Assumed source repository: `backend-api`
- Source docs root: `/docs/**`
- Central sync target: `docs/services/backend-api/synced/**`

## Platform Responsibilities

- Authenticate and authorize users
- Expose search, profile, preference, and application-tracker workflows
- Orchestrate fit-scoring and explanation requests to the Model API
- Read and persist normalized jobs and user state in PostgreSQL

## Key Dependencies And Consumers

| Type                  | System      | Why It Matters                                                 |
| --------------------- | ----------- | -------------------------------------------------------------- |
| Upstream consumer     | Frontend UI | The frontend depends on the backend for every product workflow |
| Downstream dependency | PostgreSQL  | Stores application state and normalized job data               |
| Downstream dependency | Model API   | Provides fit and explanation outputs                           |
| Upstream data source  | Scraper API | Supplies normalized job inventory used by backend features     |
