# Central Documentation Hub

This repository is the single entry point for understanding the Bisakerja platform at the system level.

It exists to do three things well:

- Explain the platform in one place for new engineers, product stakeholders, and cross-team contributors
- Aggregate service-owned documentation from the Frontend UI, Backend API, Scraper API, and Model API repositories
- Define the standards that keep distributed documentation consistent, reviewable, and maintainable

## What Is Implemented In The Foundation Layer

The foundation layer establishes the documentation baseline:

- Scope and ownership boundaries
- Information architecture and repository structure
- Naming conventions and document metadata
- Contribution and review rules
- Top-level documentation sections for overview, services, operations, standards, and references

System architecture, end-to-end data flow, and deeper service documentation are planned next and intentionally separated into subsequent milestones.

## Foundation Outcome

With the foundation in place, this repository should answer these questions clearly:

- What belongs in the central repository versus service repositories
- How documentation is organized and navigated
- Who owns each documentation domain
- How new documents must be named, tagged, reviewed, and maintained

## Overview Outcome

The overview layer adds the platform overview set that explains:

- The product problem and intended user experience
- The role of each major service in the system
- The platform's high-level architecture and service interaction model
- Database ownership and the main domain entities
- End-to-end user journeys from job discovery to application tracking

## Reader Orientation Outcome

The reader-orientation layer adds:

- Where product, overview, service, and synced docs should be entered
- How different reader roles should navigate the docs
- How onboarding and related-doc patterns reduce dead ends
