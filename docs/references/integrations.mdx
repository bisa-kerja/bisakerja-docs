# Integrations

This page tracks the major external dependencies and platform integrations that influence system behavior.

## External Job Platforms

The project brief names these primary job data sources:

- Glints
- Jobstreet
- Kalibrr
- Dealls

### Integration Concerns

- Changes in page structure or API surface
- Throttling and anti-bot protections
- Inconsistent salary, location, or requirement fields
- Duplicate listings across sources
- Public backend responses must use normalized source platform metadata and product-safe job fields, not raw provider payloads

## Model and AI Stack

The product brief references these model-related components:

- TensorFlow
- Sentence-transformer style embedding or semantic processing workflows
- FastAPI for inference serving

### Integration Concerns

- Model artifact compatibility with serving runtime
- Input schema drift between backend and model service
- Explainability output consistency
- Versioned payload and response contracts should be validated in the backend before user-facing responses are returned
- Timeout and availability failures should degrade only AI workflows, not unrelated search or tracker workflows

### Backend Upload Boundary

For CV analysis, the backend currently accepts a temporary PDF upload, stores only sanitized file metadata plus a private storage key, and forwards a backend-owned file reference to the model service. Raw CV text is not persisted by default. Temporary local files must be cleaned through a documented retention workflow rather than exposed through public URLs.

## Platform And Development Tooling

The current brief references:

- PostgreSQL
- Prisma
- Docker
- Git and GitHub

### Integration Concerns

- Schema drift between services
- Environment configuration mismatches
- Local versus deployed runtime differences

## Why This Reference Exists

These integrations matter because many user-visible incidents in Bisakerja will not come from one service alone. They often emerge at the boundaries between:

- Scraper logic and external job platforms
- Backend orchestration and model-serving contracts
- Deployment configuration and runtime behavior
