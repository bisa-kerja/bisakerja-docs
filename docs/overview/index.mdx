## What This Section Explains

This section now explains:

- the problem Bisakerja is solving for job seekers in Indonesia
- the responsibilities of Frontend UI, Backend API, Scraper API, and Model API
- the platform's end-to-end data movement from ingestion to recommendation
- the main application data domains and ownership boundaries
- concrete user-facing flows such as browsing jobs, scoring job fit, and tracking applications

## Current Documentation Boundary

The overview section remains intentionally platform-oriented:

- it defines the shared system picture
- it does not replace service-owned implementation details
- it assumes the service repositories will later contribute deeper technical docs through synchronization
